# Plan 10-01: Dark Mode for All Public Pages

## Goal
Add full dark mode support to all public-facing pages with a theme toggle that persists user preference.

## Tasks

### Task 1: Create/Update ThemeToggle Component
**File:** `apps/web/src/components/ThemeToggle.tsx`

Check if ThemeToggle already exists (used in dashboard). If so, ensure it:
- Has sun/moon icons
- Works with next-themes or similar
- Stores preference in localStorage

If creating new, implement:
```typescript
'use client';
import { useTheme } from 'next-themes';
import { Sun, Moon } from 'lucide-react';

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <button
      onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
      className="p-2 rounded-lg hover:bg-gray-100 dark:hover:bg-gray-800"
    >
      {theme === 'dark' ? <Sun className="w-5 h-5" /> : <Moon className="w-5 h-5" />}
    </button>
  );
}
```

### Task 2: Update Public Layout with Theme Provider
**File:** `apps/web/src/app/(public)/layout.tsx` or equivalent

Ensure:
1. ThemeProvider wraps public pages (if not already)
2. ThemeToggle placed in top-right of header
3. System preference respected on first load

### Task 3: Audit and Fix Public Pages

Audit ALL public pages for dark mode styles:

**Pages to check:**
- Landing page (`/`)
- Pricing page (`/pricing`)
- Features page (`/features`)
- About page (`/about`)
- Contact page (`/contact`)
- Login page (`/login`)
- Signup/Register page (`/signup` or `/register`)
- Any other public routes

**For each page, ensure:**
- Background: `bg-white dark:bg-gray-900` (or similar)
- Text: `text-gray-900 dark:text-white` for headings
- Text: `text-gray-600 dark:text-gray-300` for body
- Borders: `border-gray-200 dark:border-gray-700`
- Cards/sections: appropriate dark variants
- Buttons: dark mode hover states
- Inputs: dark mode backgrounds and borders
- Images: consider if they need dark mode variants

### Task 4: Fix Any Hardcoded Colors

Search for hardcoded colors that break dark mode:
- `bg-white` without `dark:` variant
- `text-black` or `text-gray-*` without `dark:` variant
- Inline styles with color values
- SVGs with hardcoded fills

### Task 5: Match Dashboard Color Scheme

Reference existing dashboard dark mode colors for consistency:
- Primary background: Check dashboard's dark bg
- Card backgrounds: Match dashboard cards
- Accent colors: Use same primary/accent palette
- Shadow adjustments for dark mode

## Verification

1. Toggle between light and dark mode on every public page
2. Verify no flash of unstyled content on page load
3. Check preference persists after browser refresh
4. Verify no pure white sections visible in dark mode
5. Test on mobile viewport as well
6. Verify color contrast meets accessibility standards

## Success Criteria

- [ ] ThemeToggle visible in header on all public pages
- [ ] All public pages render correctly in dark mode
- [ ] No white/unstyled sections in dark mode
- [ ] Theme preference persists in localStorage
- [ ] Colors match dashboard dark mode palette
- [ ] TypeScript compiles without errors
